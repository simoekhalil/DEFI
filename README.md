# QA Testing

This repository contains Playwright end-to-end tests that exercise the live LPAD frontend with a MetaMask wallet provided by [dappwright](https://github.com/TenKeyLabs/dappwright).

## Getting started

1. Install dependencies and browsers:
   ```bash
   npm install
   npx playwright install chromium
   ```

2. Create a `.env` file (do not commit it) with your wallet credentials:
   ```ini
   BASE_URL=https://lpad-frontend-dev1.defi.gala.com
   METAMASK_PRIVATE_KEY=your_private_key_here_without_0x_prefix
   # Optional: Custom network configuration
   # RPC_URL=
   # CHAIN_ID=
   # CHAIN_NAME=
   # CHAIN_SYMBOL=
   ```
   
   **Note:** The tests use `METAMASK_PRIVATE_KEY` to import your wallet. Never commit this file or share your private key.

3. (Optional) Pre-download the MetaMask browser extension so test runs can reuse the cached archive:
   ```bash
   npm run metamask:download
   ```

## Scripts

- `npm run test:e2e` – run the Playwright suite in headless mode.
- `npm run test:e2e:headed` – run tests with a visible browser window for debugging (Windows: use `$env:HEADLESS='false'; npm run test:e2e`).
- `npm run report` – open the HTML report generated by the previous run.
- `npm run bundle:artifacts` – archive the Playwright report and raw artifacts into `artifacts/playwright-artifacts-<timestamp>.zip`.
- `npm run metamask:download` – fetch the MetaMask extension archive into dAppwright's local cache without launching tests.

Artifacts from each run (screenshots, videos, traces, HTML report, JSON, JUnit) are written to `test-results/` and `playwright-report/`.

## CI/CD

### GitHub Actions

The tests run automatically on GitHub Actions for:
- **Push to main/master/develop branches**
- **Pull requests** to main/master/develop
- **Daily schedule** at 2 AM UTC
- **Manual trigger** via workflow dispatch

### Setting up GitHub Secrets

To run tests in CI, you need to set up the following repository secret:

1. Go to your GitHub repository
2. Navigate to **Settings** → **Secrets and variables** → **Actions**
3. Click **New repository secret**
4. Add the following secret:
   - **Name:** `METAMASK_PRIVATE_KEY`
   - **Value:** Your wallet's private key (without `0x` prefix)

#### Optional Secrets (if using custom network):
- `RPC_URL` - Custom RPC endpoint
- `CHAIN_ID` - Custom chain ID
- `CHAIN_NAME` - Custom chain name
- `CHAIN_SYMBOL` - Custom chain symbol (e.g., ETH)

### CI Features

- ✅ Runs tests in headless Chromium
- ✅ Downloads MetaMask extension automatically
- ✅ Uploads test reports and artifacts (retained for 30 days)
- ✅ Comments on PRs when tests fail
- ✅ Prevents accidental use of bundled test credentials in CI

### Viewing Test Results

After a CI run:
1. Go to the **Actions** tab in your GitHub repository
2. Click on the workflow run
3. Download the **playwright-report** or **test-results** artifacts to view detailed results and screenshots
